#  Task 3: Improve one user story with acceptance criteria.

## User Story 3.2: Update Job Status

As a mechanic or service advisor, I want to update the status of a vehicle job card in real time so that workshop staff and customers can accurately track repair progress and receive important updates throughout the repair process. 

### Improved Acceptance Criteria

1. Authorized users such as mechanics and service advisors should be able to update the job status.

2. The system should save the updated status immediately after submission.

3. Each status update should record:
    * the user who made the update
    * the timestamp of the update
    * the previous and new status

4. Customers should receive notifications for important status chnages such as:
    * Quote Awaiting Approval
    * Parts Ordered
    * Repair In Progress
    * Ready for Collection
    * Completed

5. The system should display the full job status history in chronological order.

6. Invalid or unauthorized status changes should be blocked

7. Customer-facing messages should use clear and professional wording.

8. The updated status should be visible on both the workshop dashboard and customer tracking page.
